Regras de UI aprendidas na prática

  • Sidebar de filtro fixa ao rolar precisa de position:sticky no elemento e a coluna-pai não pode ter align-self:flex-start nem align-items-start na row — isso remove o "espaço" que o sticky precisa pra funcionar.
  • Callout/highlight boxes usam fundo tintado + border: 1px solid uniforme — nunca border-left grosso (visual datado, evitar reintroduzir em cards novos).
  • Todo link de navegação entre páginas precisa usar o mesmo nome de parâmetro que a página de destino espera (ex: ppg.php só lê ?ppg=<nome>, nunca ?codigo=) — já causou bug de "botão que só recarrega a tela".
  • Menu de usuário logado (avatar + iniciais + dropdown) vive no Navbar component e lê $_SESSION diretamente — funciona em qualquer página que chame Navbar::display(), sem precisar de função extra por página.
  • <code> inline sempre com fundo cinza-claro fixo — por isso sempre defina uma cor de texto explícita nele, nunca deixe herdar do parágrafo pai (parágrafos sobre fundo escuro têm texto claro, que fica ilegível num fundo claro).
